## 3. 模型与输入输出

### 3.1 模型主结构

当前仍是共享 backbone 的 DeepONet 风格结构：

- branch 输入：几何参数 + 工况编码
- trunk 输入：空间坐标点
- field head：输出点场 `u, v, p, nut`
- scalar head：输出 `cl, cd`
- feature head：输出 `pressure_gradient_indicator, high_gradient_mask`
- surface / slice 输出：由 field 预测再结合后处理导出

### 3.2 各输出量的物理意义

- `u`：x 方向速度分量
- `v`：y 方向速度分量
- `p`：压力场
- `nut`：湍流运动粘性，表示湍流混合导致的等效扩散能力，不是密度
- `cl`：升力系数
- `cd`：阻力系数
- `pressure_surface`：翼型表面压力分布
- `cp_surface`：表面压力系数，表示表面压力相对自由来流动压的无量纲分布
- `slice`：指定直线截面上的 `u / v / p / nut` 分布
- `high_gradient_mask`：高梯度区域指示，用于辅助识别压力或流场快速变化区

### 3.3 为什么第四通道是 `nut` 不是 `rho`

AirfRANS 当前主线是 2D、不可压缩、亚声速 RANS。该数据条件下，真实可用主字段更接近：

- `velocity`
- `pressure`
- `turbulent viscosity`

因此第 4 通道更合理地定义为 `nut`，而不是 `rho`。

- `rho` 是密度，更多属于可压缩流主变量
- `nut` 是湍流模型里的涡粘性/运动粘性，是当前数据真实支持的监督量

## 9. 结果分析

### 9.1 当前已经打通的真实输出

以下输出已真实打通并参与了主流程：

- `u / v / p / nut`
- `cl / cd`
- `pressure_surface`
- `cp_surface`
- `slice u / v / p / nut`
- `high_gradient_mask`

### 9.2 当前表现较好的部分

- `cd` 误差较低，说明标量阻力趋势已有一定可用性
- `cp_surface` 相比 bug 修复前已有明显恢复，不再塌成接近 0
- `slice` 输出可稳定导出，说明 field 到 analysis 的链路已经贯通
- `high_gradient_mask` 至少已有一定区分能力，`IoU` 达到 `0.335`

### 9.3 当前仍然偏弱的部分

- `field_rmse` 仍偏高，说明全场细节拟合还不够强
- `pressure_surface_rmse` 很大，说明 raw pressure 重建尺度仍比较敏感
- `pressure_gradient_indicator` 的 `F1/IoU` 为 `0`，当前 feature head 在这一分支上基本学成了全负预测
- physics loss 虽然已接入 `nut` 路径，但目前更像正则项，还没有充分转化为表面压力和 feature 上的稳定收益

### 9.4 对 physics loss 的判断

这轮 physics loss 是有实际生效的，不再是旧版本那样因为第 4 通道不是 `rho` 而直接返回 `0`。但从结果看：

- 它对训练稳定性和整体场约束有帮助
- 但并没有自动把 `pressure_surface` 或 `pressure_gradient_indicator` 一并拉好

这说明当前 physics 项的强度、形式和采样方式后面仍需要继续调。

## 11. 后续优化建议

建议优先做以下几项：

1. 调整 physics loss 权重和采样策略，不只在 query points 上约束，也可增加靠近表面和尾迹区的物理点密度。
2. 对 `pressure_gradient_indicator` 使用更稳健的伪标签策略，避免类别极不平衡导致模型学成全负。
3. 强化 surface 分支，给 `surface_points` 增加更直接的 head 或更高权重监督，继续压低 `cp_surface_rmse`。
4. 对 `pressure_target_mode=cp_like` 的数据路径继续统一量纲处理，避免 raw pressure 与 cp 两套尺度互相干扰。
5. 将 `slice` 定义从当前默认抽样扩展为更贴近翼型分析的多截面集合，例如前缘、最大厚度、尾迹线。

## 12. 本轮结论

本轮训练已经完成以下目标：

- 使用全量 AirfRANS heavy 数据做了更合理的 `700 / 150 / 150` 切分
- 按 `1024 query points / 256 surface points / latent_dim=256 / 30 epoch` 重新完成训练
- 将 physics loss 改成了可兼容 `nut` 的版本，并成功参与训练
- 完成了测试评估、图像导出和分析 bundle 导出
- 修复了此前 `surface Cp` 预测接近 0 的核心 bug

当前模型已经可以作为一个可分析输出的 2D AirfRANS 代理基线继续往下优化，但表面压力重建、feature 分支稳定性和物理约束收益仍有明显提升空间。
